# gargantua UI

Next.js admin + chat console for the gargantua backend.  Builds as a
fully static export (`out/`), served by FastAPI under `/` in
production; served directly by `next dev` in development.

## Quickstart

```bash
cd ui
pnpm install
cp .env.example .env.local       # tweak NEXT_PUBLIC_API_BASE_URL if needed
pnpm dev                         # http://localhost:3000
```

You'll need the backend running too:

```bash
# in repo root
uvicorn gargantua.main:app --host 0.0.0.0 --port 7777
```

The UI talks to `NEXT_PUBLIC_API_BASE_URL` (default
`http://localhost:7777` in `.env.example`).  CORS for `:3000` is on
by default in dev ‚Äî see `CORS_ALLOW_ORIGINS` in `.env.example` at the
repo root if you've tightened it.

## Scripts

| Command | Purpose |
|---|---|
| `pnpm dev` | Dev server with hot reload |
| `pnpm build` | Production static export ‚Üí `out/` |
| `pnpm start` | Serve `out/` with `serve` on :3000 (mirrors prod) |
| `pnpm typecheck` | `tsc --noEmit` |
| `pnpm lint` | `next lint` (ESLint + Next rules + prettier disable) |
| `pnpm test` | Vitest unit + component tests |
| `pnpm test:e2e` | Playwright E2E (stubs all API calls ‚Äî no backend needed) |
| `pnpm codegen` | Regenerate `lib/api/types.generated.ts` from the running backend's `/openapi.json` |

## Auth model

* Tokens come from `POST /auth/login` as `{access_token, refresh_token, expires_in}`.
* Stored in `localStorage` (XSS trade-off is acknowledged in the
  plan: `gargantua-ui-ef823e.md` open-items ¬ß1).
* Every request goes through `apiFetch` in `lib/api/client.ts`,
  which attaches `Authorization: Bearer ...` and, on a 401, tries a
  single `/auth/refresh` + retry before clearing tokens.
* Page-level gating is client-side via `<AuthGuard>` /
  `<GuestOnly>` ‚Äî static export has no middleware layer at request
  time, so we can't use `middleware.ts`.

## Type generation

`lib/api/types.ts` hand-rolls the small auth surface; everything else
is generated by `openapi-typescript` from the backend's `/openapi.json`
into `lib/api/types.generated.ts`:

```bash
# backend must be running at NEXT_PUBLIC_API_BASE_URL (or default :7777)
pnpm codegen
```

Re-run after any backend schema change.

## Testing

Two layers:

* **Vitest** (`tests/`) ‚Äî module-level unit tests with `jsdom`.  Run
  `pnpm test` (or `pnpm test:watch`).  No backend needed; `fetch` is
  stubbed via `vi.stubGlobal`.
* **Playwright** (`e2e/`) ‚Äî full browser, network stubbed via
  `page.route`.  `pnpm test:e2e` builds the production bundle first
  via `webServer.command` (`pnpm build && pnpm start`).
